# 公平网约车治理 APP — Fair Dispatch OS

> 让乘客知道为什么等，让司机知道为什么没单。

---

## 🌌 这是什么

一个用**暗黑宇宙粒子风格**构建的网约车治理原型——纯黑画布上，数千粒子聚集成出租车轮廓，Plum Violet 脉冲是唯一的色彩权威。在这里，每一次派单都可解释，每一秒等待都可追溯，每一个异常都可申诉。

**不是"正在为你呼叫司机"，而是告诉你订单去了哪里。**

---

## ❌ 它在解决什么

| 痛点 | 现状 | Fair Dispatch OS |
|------|------|------------------|
| 打不到车 | 只看到"等待中" | 展示排队位置、附近司机、压单风险 |
| 接不到单 | 不知道为什么没派给我 | 记录可服务但未派单的异常证据 |
| 压单 | 订单消失在黑箱 | 每秒都有事件日志，超阈值即预警 |
| 派单不公 | 算法权重是谜 | 每一单给出评分解释，权重可调可审 |
| 一单两买 | 同一单卖两次 | 订单指纹跨平台比对，自动标记冲突 |
| 民生时段 | 订单外流、无保障 | 早晚高峰/医院/学校订单强制加分 |
| 申诉无门 | 投诉石沉大海 | 证据链驱动的闭环申诉流程 |

## ✨ 核心功能

### 🔍 乘客透明叫车
不再盯着空白屏幕——看到排队第几位、附近有多少可服务司机、为什么还没派单、是否被压单。

### 🚖 司机公平接单
证明自己在线、空闲、在服务半径内。如果订单存在但司机没收到单，系统自动生成"未派单异常"证据。

### ⚖️ 公平派单引擎
综合五维评分：距离（32%）+ 乘客等待（26%）+ 司机空闲（20%）+ 服务均衡（15%）+ 服务质量（7%）。民生保障订单额外加分。**每一单都给出"为什么派给 TA"。**

### 🔗 订单证据链
从创建→入池→派单→接驾，全链路事件留痕。轻量 hash chain 演示不可篡改存证思路，支持一键校验和模拟篡改。

### 🛡️ 防一单两买
每个订单生成唯一指纹，跨平台（A/B/C）自动比对时间窗口、价格、路线，识别重复售卖。

### 🏥 民用时段保障
早晚高峰、夜间、恶劣天气、医院/学校场景自动启用保障策略，保障订单获得优先派单和最低派单率。

### 📊 监管审计看板
压单率、公平派单指数、平均等待、一单两买风险——关键指标一屏呈现，异常订单证据链表格可追溯。

### 📋 异常申诉闭环
乘客压单申诉 / 司机未派单申诉 / 重复售卖申诉 → 已提交 → 取证中 → 平台待解释 → 已裁定，每一步留痕。

## ⌨️ 交互体验

- **粒子星座引擎** — 150+ 车形粒子 + 120 漂浮粒子，鼠标推散回弹
- **Toast 通知** — 每次操作右上角弹出紫光通知
- **滚动动画** — Section 进入视口时淡入上浮
- **鼠标光晕** — Plum 径向渐变跟随光标
- **地图探查** — 点击网格查看该位置的司机/订单信息
- **自动微动** — 司机位置每 8 秒微移，地图始终"活着"
- **键盘快捷键** — `D` 派单 · `A` 审计 · `M` 地图推进 · `V` 校验链 · `1/2/3` 切换角色 · `?` 帮助

## 🧭 推荐演示路径

1. 打开页面 → 粒子从四方聚拢成出租车
2. **角色工作台** → 切换乘客/司机/监管三种视角
3. **推进 30 秒** → 观察司机位置与可服务状态变化
4. **运行审计** → 监管指标刷新、事件流增长、派单日志生成
5. **校验证据链** → 确认 hash chain 完整
6. **模拟篡改** → 再校验，观察断裂警告
7. **跨平台指纹比对** → 识别疑似一单两买
8. **发起申诉** → 推进处理状态，每步留痕
9. **调整算法权重** → 再派单，观察评分变化
10. 按 `?` 查看全部快捷键
