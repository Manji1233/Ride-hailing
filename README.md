# 公平网约车治理 APP 原型

这是一个本地静态 Web App 原型，已按 `DESIGN.md` 的黑白交通 kiosk / Uber 风格重做视觉设计，用来讨论并演示以下网约车治理问题：

- 用户打不到车，司机却接不到单
- 压单
- 派单不公平
- 民用时段订单保障不足，订单可能被转卖给其他平台
- 一单两买 / 多平台冲突
- 异常申诉缺少证据链
- 派单算法权重不透明

## 打开方式

直接双击打开：

```text
D:\AI\网约车\index.html
```

或用浏览器打开该文件。

本原型不需要安装依赖，不需要启动后端。

## 当前包含的页面模块

1. **乘客透明叫车**：展示排队位置、等待解释、压单风险。
2. **司机公平接单**：展示司机在线、距离、空闲、服务半径和公平补偿分。
3. **角色工作台**：可切换乘客、司机、监管三种视角。
4. **地图与司机实时位置模拟**：用城市网格模拟乘客、目的地、司机、热区和可服务状态。
5. **监管审计看板**：展示压单率、公平派单指数、平均等待、一单两买风险。
6. **后端订单流日志 Mock**：展示订单创建、入池、派单尝试、司机通知、风险预警等事件。
7. **不可篡改审计链演示**：用轻量 hash chain 展示事件存证思路，并支持校验和模拟篡改。
8. **跨平台订单指纹比对**：模拟平台 A/B/C 上的相似订单，识别疑似一单两买。
9. **异常申诉流程**：支持乘客、司机、一单两买申诉，并推进处理状态。
10. **公平派单模拟**：根据距离、等待、司机空闲、服务均衡和服务质量计算派单得分。
11. **派单算法配置后台 Mock**：可调整派单权重和民生时段加分，观察派单解释变化。
12. **ASCII 产品蓝图**：用于讨论产品结构。

## 推荐演示路径

1. 打开 `index.html`。
2. 在“角色工作台”切换乘客、司机、监管视角。
3. 点击“推进 30 秒”，观察司机位置与可服务状态变化。
4. 点击“运行审计”，观察监管指标、事件流、派单日志变化。
5. 点击“校验证据链”，查看 hash chain 校验结果。
6. 点击“模拟篡改”，再校验证据链，观察异常提示。
7. 查看“跨平台订单指纹比对”，识别疑似重复售卖订单。
8. 点击乘客/司机/重复售卖申诉，推进申诉状态。
9. 调整“派单算法配置后台 Mock”中的权重，再点击“模拟下一单派发”。
